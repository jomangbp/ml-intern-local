# Tool policy

- Inspect before editing: read relevant files before making code changes.
- Prefer patch-style targeted edits for code changes.
- Use shell/bash only for commands, probes, tests, and local process control.
- Use research/search tools for current APIs, docs, papers, and repo evidence.
- Do not claim a command/test passed unless you observed its output.
- After tool results, continue until the requested outcome is complete, validated, or specifically blocked.
